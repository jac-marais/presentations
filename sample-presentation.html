Context

What This Is

A personal repository for hosting self-contained HTML presentations via GitHub Pages. Each presentation is a single file: no build step, no server, no dependencies.

Single File

Each deck is one .html file with inline CSS, JS, and SVG. Open it with a double-click.

Private Pages

Hosted on GitHub Pages with private visibility. Only people with repo access can view the deployed site.

Version Controlled

Every revision is tracked in git. Compare versions, roll back changes, review diffs on any slide.

Capabilities

Built-in Features

  • N
    Keyboard Navigation
    Arrow keys and spacebar move between slides. Scroll-snap keeps alignment pixel-perfect.
  • A
    Scroll Animations
    Elements fade in with staggered timing as each slide enters the viewport via IntersectionObserver.
  • D
    Dark Theme
    Consistent color system with CSS custom properties. Looks sharp on projectors and screens alike.
  • S
    Inline SVG Diagrams
    Vector diagrams render crisply at any resolution. No external image files needed.
  • R
    Responsive Layout
    Cards and grids adapt to screen size. Nav dots reposition on mobile viewports.
Architecture

How It Works

The deployment pipeline: author locally, push to GitHub, Pages serves the result.

Example

Adding a New Slide

Each slide is a <section> element. Add content, push, and it is live.

presentation.html 
<!-- Add a new slide -->
<section class="slide" data-title="My Slide">
  <div class="slide-inner">
    <span class="kicker anim">Section</span>
    <h1 class="anim">Slide Title</h1>
    <p class="anim">Your content here.</p>
  </div>
</section>
